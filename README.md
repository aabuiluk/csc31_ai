# csc31_ai

Невеликий термінальний бот: ви вводите питання в консоль, відповідь формує модель **gpt-4o-mini-2024-07-18** (OpenAI API).

## Вимоги

- Python 3.10 або новіше
- Обліковий запис OpenAI та [API key](https://platform.openai.com/api-keys)

## Встановлення

1. Клонуйте репозиторій або відкрийте папку проєкту.

2. (Рекомендовано) створіть віртуальне середовище та активуйте його:

   ```bash
   python3 -m venv .venv
   source .venv/bin/activate
   pip install --upgrade pip
   ```

3. Встановіть залежності:

   ```bash
   pip install -r requirements.txt
   ```

## Токен у `.env`

1. Скопіюйте приклад конфігурації:

   ```bash
   cp .env.example .env
   ```

2. Відкрийте файл `.env` у текстовому редакторі.

3. Замініть значення `OPENAI_API_KEY` на ваш реальний ключ з [OpenAI](https://platform.openai.com/api-keys). Рядок має виглядати так:

   ```env
   OPENAI_API_KEY=sk-...
   ```

   Файл `.env` уже в `.gitignore` — не комітьте його в git.

Детальніші підказки див. у коментарях у [.env.example](.env.example).

## Запуск

З активованим venv (якщо використовуєте):

```bash
python bot.py
```

Бот> Бульбашкове сортування — це простий алгоритм сортування, який працює шляхом повторного проходження через масив і порівняння сусідніх пар елементів, обмінюючи їх в разі потреби. Цей процес повторюється, поки масив не буде відсортовано. Ось реалізація бульбашкового сортування на Python:

```python
def bubble_sort(arr):
    n = len(arr)
    # Проходимо через всі елементи масиву
    for i in range(n):
        # Останні i елементів вже відсортовані, тому їх можна ігнорувати
        for j in range(0, n-i-1):
            # Якщо поточний елемент більший за наступний, обмінюємо їх
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
    return arr

# Приклад використання
unsorted_array = [64, 34, 25, 12, 22, 11, 90]
sorted_array = bubble_sort(unsorted_array)
print("Відсортований масив:", sorted_array)
```

Код вище реалізує бульбашкове сортування, яке сортуює масив `unsorted_array` і виводить відсортований масив. Алгоритм має часову складність O(n^2) в гіршому та середньому випадках, що робить його неефективним для великих масивів.
